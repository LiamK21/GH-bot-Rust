Issue:
<issue>
When parsing files, cleanup "." and ".." path components
For some files (luckily rarely and only for header files), in the reports we have things like "third_party/aom/aom_util/./aom_thread.h".
We can't just cleanup the paths at the end, as they appear both in the "third_party/aom/aom_util/aom_thread.h" and "third_party/aom/aom_util/./aom_thread.h" version in the same report.
We can clean them up when merging the results: https://github.com/marco-c/grcov/blob/aae34d4a234aeac63643676617f54b99c63e5601/src/lib.rs#L76.
</issue>
