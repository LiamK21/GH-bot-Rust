Issue:
<issue>
Wrong LLOC computation for the Rust language
Initially we assumed that **ALL** Rust expressions were statements. For example:

Accessing a field --> `struct.field`
Parenthesized expressions --> `(42 + 0)`
Tuple expressions ---> `(0, 42)`
Array expressions ---> `[0; 42]`
Functions and macros when they are not used as statements ---> `if foo() {}` or `if foo!() {}`

This approach leaded to have an high and wrong `LLOC`  for the `Rust` language. It would be helpful to remove these statements and consider only the effective ones.
</issue>
