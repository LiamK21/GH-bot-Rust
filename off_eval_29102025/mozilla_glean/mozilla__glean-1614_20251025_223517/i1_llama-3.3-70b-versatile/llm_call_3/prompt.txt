<issue>
RLB Race Condition - `was_initialize_called` is not a sufficient guard for `with_glean`/`with_glean_mut`
As seen in the backout for bug 1672455, there's a race condition in calling `set_source_tags` after `initialize` in the RLB.

It seems to be that `set_source_tags` guards its use of `with_glean_mut` on `was_initialize_called`. Which is odd, because the global glean isn't set until later (perhaps arbitrarily later, since it's done on a different thread) during `glean_core::setup_glean`.

It seems to me as though we need a later guard for `was_glean_set_up` or something to guard uses of `global_glean`. Either that, or instead of trying to flag some state we should query it directly by using `global_glean().is_some()` instead of `was_initialize_called()`.
</issue>
