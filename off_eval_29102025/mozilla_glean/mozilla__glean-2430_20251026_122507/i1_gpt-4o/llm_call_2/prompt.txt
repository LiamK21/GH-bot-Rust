Glean iOS UniffiInternalError.rustPanic("No database found") crash during tests
While upgrading Glean iOS to 52.2.0 from 51.8.3, we experienced a crash in unit tests when using the `Glean.shared.resetGlean(clearStores: true)` API. When we set this to `false` with `Glean.shared.resetGlean(clearStores: false)` then this fixes the crash, but makes tests harder to write since they can depend on previous states (since store isn't reset).
