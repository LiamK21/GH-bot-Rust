Issue:
<issue>
Replace use of lazy_static with once_cell
Initially we started using `lazy_static` for all our globals, mostly because our dependencies used it anyway.
We now are _also_ using `once_cell` for some of these globals.
We should unify it to a single crate.

Reasons for once_cell: It avoids a macro and instead uses plain Rust code, which IMO is desirable.

Note: our dependencies (notably rkv and ffi-support) still depend on lazy_static, so it won't go away from our dependency tree.
</issue>
