Issue:
<issue>
Event max_events overflow doesn't trigger upload right away
When an event is recorded and the `max_events` limit is reached an `events` ping is submitted, but this doesn't trigger the uploader.
The ping thus can stick around for a while until the uploader runs the next time.
</issue>
