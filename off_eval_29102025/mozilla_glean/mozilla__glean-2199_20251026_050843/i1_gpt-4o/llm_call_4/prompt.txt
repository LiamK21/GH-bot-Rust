Issue:
<issue>
Bump the limit for the URL metric type to 8k and support truncation
[Original GH Issue](https://github.com/mozilla/glean.js/issues/1508)

Glean imposes limits on certain metric types, e.g.
- max length of 100 for strings: https://mozilla.github.io/glean/book/reference/metrics/string.html#limits
- max length of 2048 for URLs: https://mozilla.github.io/glean/book/reference/metrics/url.html#limits
- and so on

Glean.js enforces these limits on the client side, so it would be great if **these constants could be exported** so that they can be checked before setting metrics. This would allow the client to e.g. skip a metric, or truncate it, without producing an `invalid_overflow` error.
</issue>
