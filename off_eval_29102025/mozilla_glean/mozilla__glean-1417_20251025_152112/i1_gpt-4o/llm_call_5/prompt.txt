Issue:
<issue>
RLB traits methods cannot take mutable references to self
Generated metrics instances in Rust are public statics, so in order to not require `unsafe{}` around all uses of a `pub static mut`, we must make operations on the instances not require a mutable reference.

The only current trait that does not satisfy this is Timing Distribution. This bug will fix that.
</issue>
