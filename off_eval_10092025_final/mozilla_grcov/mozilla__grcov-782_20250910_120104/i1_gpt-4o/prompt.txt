 0.8.5 always reports 0% in coverage.json 
It seems the coverage.json file generated for shields.io is broken in 0.8.5, as it now always reports 0% coverage. The HTML file created is still correct and shows the right coverage stats. Reverting to 0.8.4 fixes it, and the project then shows the correct percentage again.

With 0.8.5:
```json
{"schemaVersion":1,"label":"coverage","message":"0%","color":"red"}
```

And 0.8.4:
```json
{"schemaVersion":1,"label":"coverage","message":"24%","color":"red"}
```

