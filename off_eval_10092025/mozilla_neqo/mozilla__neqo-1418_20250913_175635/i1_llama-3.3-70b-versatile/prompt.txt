Check WebTransport request header (server-side)
The spec says:

> In order to create a new WebTransport session, a client can send an
   HTTP CONNECT request.  The :protocol pseudo-header field ([[RFC8441](https://datatracker.ietf.org/doc/html/rfc8441)])
   MUST be set to webtransport.  The :scheme field MUST be https.  Both
   the :authority and the :path value MUST be set; those fields indicate
   the desired WebTransport server.  An Origin header [[RFC6454](https://datatracker.ietf.org/doc/html/rfc6454)] MUST be
   provided within the request.

Currently e only check for the "authority" header.
</issue>
