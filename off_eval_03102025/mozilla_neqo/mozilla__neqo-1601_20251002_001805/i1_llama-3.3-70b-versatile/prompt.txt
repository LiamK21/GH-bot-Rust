Issue:
<issue>
Don't format packets for dumping if qlog has not been configured to log
If the log::max_level() is Debug or Info, we'll spend the time to format packets in dump_packet() even if the qdebug!() will end up as a no-op, since we haven't configured a file for logging.

This can cost us 10% of a core when downloading over h3 on a high end machine.

The log level is a shared resource, so it can get bumped by other modules: https://bugzilla.mozilla.org/show_bug.cgi?id=1874215
</issue>
